## 1. 游戏概述

**地下城生存**是一款俯视角（Top-Down）的 Roguelite 生存射击游戏。玩家扮演战士或法师，在被诅咒的地下城中抵御一波又一波怪物的进攻，击败最终 Boss 获得胜利。

- **胜利条件**：通过全部 4 个波次并击杀终极 Boss「恐惧领主」（Dreadlord Boss）
- **失败条件**：玩家生命值归零
- **游戏时长**：约 5-10 分钟/局

## 2. 设计理念

### 2.1 核心设计原则

| 原则 | 阐述 |
|------|------|
| **波次递进式挑战** | 游戏分为 4 个波次，每波难度递增，给玩家清晰的阶段性目标和成长节奏 |
| **Roguelite 升级成长** | 通过击杀敌人获取经验、升级，从随机池中选择强化能力，每局体验不同 |
| **双角色差异化** | 战士近战冲刺 vs 法师远程范围技能，提供不同的操作体验和策略深度 |
| **简洁而可读的信息** | "场上同时敌人上限 + 远程敌人上限"的双重限制，确保战斗可管理、不混乱 |
| **清场过渡体验** | 每波结束后弹出升级选择 + 短暂休息期，给玩家喘息和策略规划的空间 |

### 2.2 目标玩家

- 喜欢短局、即开即玩的休闲/中度玩家
- 对 Roguelite 和弹幕射击有兴趣的玩家
- 偏好策略选择而非纯反应速度的玩家

## 3. 游戏流程

```
┌──────────────┐
│   主菜单      │
│ (选择角色)    │
└──────┬───────┘
       ↓
┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│   Wave 1     │ → │   Wave 2     │ → │   Wave 3     │ → │   Wave 4     │
│ (难度 0)     │    │ (难度 1)     │    │ (难度 2)     │    │ (难度 3)     │
│ 生成 8 只怪  │    │ 生成 12 只怪 │    │ 生成 18 只怪 │    │ 生成 24 只怪 │
│ 场上 ≤ 2 只  │    │ 场上 ≤ 4 只  │    │ 场上 ≤ 6 只  │    │ 场上 ≤ 6 只  │
└──────┬───────┘    └──────┬───────┘    └──────┬───────┘    └──────┬───────┘
       ↓                   ↓                   ↓                   ↓
  [清场完毕]           [清场完毕]           [清场完毕]           [清场完毕]
       ↓                   ↓                   ↓                   ↓
  🔼升级选择             🔼升级选择           🔼升级选择           🔼升级选择
  等待 3 秒              等待 3 秒            等待 3 秒            ┌──────────────┐
       ↓                   ↓                   ↓                │  Boss 登场！ │
  进入 Wave 2            进入 Wave 3          进入 Wave 4        │  恐惧领主     │
                                                               └──────┬───────┘
                                                                      ↓
                                                               [击杀 Boss]
                                                                      ↓
                                                               🏆 胜利！
```

### 3.1 波次推进逻辑

1. 当前波次所有预定怪物生成完毕
2. 场上不再生成新怪（`isSpawningSmall = false`）
3. 玩家击杀场上所有残余怪物
4. 系统自动弹出 1 次升级选择（时间暂停）
5. 选择升级后恢复，等待 3 秒自由活动时间（玩家可移动但无怪物）
6. 下一波次开始生成怪物

## 5. 敌人系统

### 5.1 敌人总览

游戏共有 **5 种敌人类型**（4 种普通小怪 + 1 种 Boss）：

| 名称 | 攻击方式 | 基础HP | 移速 | 触伤 | 弹伤 | 攻击范围 | 冷却 | 经验 |
|------|---------|--------|------|------|------|---------|------|------|
| 蝙蝠 (Bat) | 近战追踪 | 30 | 4.0 | 8 | - | - | - | 12 |
| 史莱姆 (Slime) | 近战追踪 | 110 | 2.5 | 10 | - | - | - | 10 |
| 蘑菇魔 (Mushroom Fiend) | 远程单发毒球 | 85 | 2.7 | 12 | 8 | 4.1 | 1.55s | 14 |
| 水晶灵 (Crystal Wisp) | 远程单发冰刺 | 45 | 3.6 | 9 | 9 | 4.8 | 1.25s | 16 |
| **恐惧领主 (Dreadlord Boss)** | Slam 范围重击 | 650 | 1.7 | 22 | Slam 30 | 2.35 | 2.4s | 80 |

### 5.2 精英机制（Elite）

每个波次有固定概率生成精英敌人（触发条件基于波次难度等级，不再基于游戏时间）：

| 波次 | Wave 1 | Wave 2 | Wave 3 | Wave 4 |
|------|--------|--------|--------|--------|
| 精英概率 | 10% | 15% | 20% | 25% |

| 精英加成 | 倍数 |
|---------|------|
| HP | ×1.8 |
| 伤害 | ×1.5 |
| 速度 | ×0.91 |
| 经验 | ×2 |
| 体型 | 略大 |
| 颜色 | 特殊配色 |

### 5.3 Boss：恐惧领主

- 仅在 Wave 4 清场后出现
- 拥有 5 种精灵动画：待机、蓄力、重击、恢复、4帧行走循环
- 行为模式：追踪玩家 → 进入范围后蓄力 → Slam 范围攻击 → 恢复 → 重复
- 击败 Boss 即胜利

### 5.4 远程敌人限制

为防止远程弹幕过多导致不公平，场上最多同时存在的远程类型敌人（蘑菇魔 + 水晶灵）数量受限：

| 波次 | 远程上限 |
|------|---------|
| Wave 1 | 1 |
| Wave 2 | 2 |
| Wave 3-4 | 3 |

超过上限时，新生成的远程敌人会被强制替换为近战类型。

## 6. 波次系统

### 6.1 难度递进

| | Wave 1 | Wave 2 | Wave 3 | Wave 4 |
|---|---|---|---|---|
| **难度等级** | 0 | 1 | 2 | 3 |
| **生成总量** | 8 只 | 12 只 | 18 只 | 24 只 |
| **场上上限** | 2 只 | 4 只 | 6 只 | 6 只 |
| **远程上限** | 1 只 | 2 只 | 3 只 | 3 只 |
| **HP 倍率** | 1.0× | 1.2× | 1.5× | 1.8× |
| **伤害倍率** | 1.0× | 1.1× | 1.25× | 1.4× |
| **速度倍率** | 1.0× | 1.05× | 1.12× | 1.2× |
| **生成间隔** | 2.0s | 1.7s | 1.35s | 1.1s |
| **Elite 概率** | 10% | 15% | 20% | 25% |

### 6.2 波次过渡

每波清场后：
1. 弹出升级选择面板（时间暂停）
2. 玩家从 3 个随机强化中选择 1 个
3. 时间恢复，等待 3 秒自由活动
4. 下一波开始出怪

## 7. 升级系统

### 7.1 升级机制

- 击杀敌人获得经验（EXP）
- 经验达到阈值后自动升级（等级公式：`EXP需求 = 20 + 当前等级 × 15`）
- 每次升级弹出 3 选 1 强化面板（时间暂停）
- 波次过渡时额外提供 1 次强化选择机会

### 7.2 升级池

共 **21 种升级**，分为 4 个类别：

#### 通用升级（Common，两个角色共享）

| 升级名称 | 效果 |
|---------|------|
| 普攻伤害 +10 | 普通攻击伤害提升 10 点 |
| 技能伤害 +5 | 技能伤害提升 5 点 |
| 最大生命 +20 | 生命上限 +20 并恢复 20 生命 |
| 移动速度 +10% | 移动速度 ×1.1 |
| 普攻冷却 -10% | 普攻冷却 ×0.9 |
| 生命恢复 +40 | 立即恢复 40 生命 |
| 技能冷却 -10% | 技能冷却 ×0.9 |

#### 法师专属升级（Mage）

| 升级名称 | 效果 |
|---------|------|
| 奥术雨扩大 | 奥术雨范围 +0.6 |
| 奥术雨延长 | 奥术雨持续时间 +1s |
| 奥术雨加速 | 奥术雨触发间隔 ×0.8 |
| 霜冻法阵 | 奥术雨减速效果从 25% 提升至 40% |
| 魔力涌动 | 技能冷却 ×0.82 |

#### 战士专属升级（Warrior）

| 升级名称 | 效果 |
|---------|------|
| 剑气扩大 | 剑气弹道尺寸 ×1.4 |
| 突进回血 | 突进斩每次命中恢复 3 HP |
| 突进冷却 | 技能冷却 ×0.75 |
| 大地裂斩 | 突进斩结束时引发范围爆裂 |
| 战斗狂热 | 突进命中后 2s 内攻速翻倍 |

#### 稀有升级（Rare，角色专属稀有）

| 角色 | 升级名称 | 效果 |
|------|---------|------|
| 法师 | 火球分裂 | 火球术以 3 发小角度散射 |
| 战士 | 剑气穿透 | 剑气可贯穿敌人 |

### 7.3 升级策略设计

- **升级随机性**：每次升级从所有可用升级中随机抽取 3 个，增加重玩价值
- **专属升级**：部分升级仅对应特定角色出现，保证角色玩法差异
- **稀有稀缺性**：稀有升级首次出现后不再重复（FireballSplit 自动移除、SwordWavePierce 自动移除）
- **数值平衡**：所有数值加成采用累乘/累加，上限控制（冷却不低于 0.08s/0.5s）

## 8. 操作与控制

### 8.1 键位配置

| 操作 | 键位 |
|------|------|
| 移动 | W/A/S/D 或 ↑←↓→ |
| 瞄准 | 鼠标移动（指针方向 = 攻击方向） |
| 普攻 | 鼠标左键（按住连发） |
| 技能 | E 键 |
| 暂停 | Esc |
| 无敌开关 | UI 按钮（调试用） |

### 8.2 操作反馈

- **普攻蓄力动画**：攻击前短暂压扁拉伸，增强打击感
- **移动动画**：人物随移动速度有呼吸缩放效果
- **受击反馈**：屏幕震动 + 红色闪烁 + 伤害数字飘出
- **技能特效**：战士突进斩有多层拖尾、幻影、火花效果

## 10. 视觉与特效

### 10.1 场景设计

- **黑暗奇幻主题**：地下城废墟风格，深色地面配以神秘符文和魔法阵装饰
- **障碍物系统**：地图上分布祭坛、石柱等障碍物，敌人和玩家受障碍物阻挡
- **墙体边界**：四面城墙限制移动范围（24×14 的竞技场）

### 10.2 特效系统

| 特效类型 | 说明 |
|---------|------|
| 圆形冲击波 | 多种颜色/尺寸，用于攻击命中、死亡爆炸等 |
| 伤害数字 | 浮空飘出，红色（玩家受伤）/ 金色（敌人受伤） |
| 拖尾效果 | 战士突进斩的多层残影和火花 |
| 临时精灵 | 带有自动销毁和淡出缩放的临时视觉元素 |
| 屏幕震动 | 击败敌人、释放技能时的相机抖动 |

### 10.3 精灵素材

项目使用像素风格精灵（Pixel Art），包含角色、敌人、弹道、地面装饰等多种类型。

## 11. 技术架构

### 11.1 脚本结构

```
Assets/Scripts/
├── Core/
│   ├── GameManager.cs        (核心控制器：波次、难度、胜负判定)
│   ├── CharacterStats.cs     (角色属性数据类)
│   ├── CharacterType.cs      (角色枚举)
│   ├── GameData.cs           (静态数据：角色初始值、敌人精灵、升级池)
│   ├── GameUtility.cs        (工具类：坐标约束、鼠标位置)
│   ├── CameraController.cs   (相机跟随)
│   ├── Obstacle.cs           (障碍物标记组件)
│   └── SceneLoader.cs        (场景加载)
├── Player/
│   ├── PlayerController.cs   (玩家输入、移动、攻击、技能)
│   ├── PlayerHealth.cs       (玩家生命值、无敌)
│   └── PlayerHealthBar.cs    (玩家血条 UI)
├── Enemy/
│   ├── EnemyController.cs    (敌人 AI、攻击行为、死亡处理)
│   ├── EnemySpawner.cs       (敌人生成器、波次管理)
│   └── EnemyHealthBar.cs     (敌人血条 UI)
├── Combat/
│   ├── Projectile.cs         (弹道行为)
│   ├── ArcaneRainArea.cs     (奥术雨范围技能)
│   ├── CombatEffectFactory.cs(特效工厂)
│   ├── DamageTextEffect.cs   (伤害数字)
│   └── TemporaryEffect.cs    (临时特效)
├── Upgrade/
│   ├── UpgradeManager.cs     (升级管理器：队列、选择、应用)
│   └── UpgradeOption.cs      (升级选项数据、枚举)
└── UI/
    ├── GameUI.cs             (游戏内 HUD 构建和刷新)
    ├── MainMenuUI.cs         (主菜单)
    └── CharacterSelectUI.cs  (角色选择)
```

### 11.2 核心设计模式

| 模式 | 应用场景 |
|------|---------|
| **单例 (Singleton)** | `GameManager.Instance` 全局唯一实例 |
| **策略 (Strategy)** | 敌人攻击类型用枚举区分不同 AI 行为 |
| **工厂 (Factory)** | `CombatEffectFactory` 统一创建特效 |
| **观察者** | `OnEnemyKilled → TryAdvanceWave` 回调链 |
| **协程** | 波次过渡等待、攻击动画、屏幕震动等 |
